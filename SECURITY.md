# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Plantegia, please report it responsibly.

**Do not open a public issue for security vulnerabilities.**

Instead, email **builder@plantegia.com** with:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Fix timeline**: Depends on severity, typically 1-4 weeks

## Scope

This policy applies to:
- The Plantegia web application (plantegia.com)
- This open source repository

Out of scope:
- Third-party services (Firebase, Vercel)
- Social engineering attacks
- Denial of service attacks

## Recognition

We appreciate responsible disclosure. Contributors who report valid vulnerabilities will be acknowledged in our release notes (unless they prefer to remain anonymous).
